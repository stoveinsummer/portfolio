# 저장소 개인정보·보안 기준

이 저장소와 GitHub Pages는 공개 공간이다. 커밋한 내용은 삭제하더라도 Git 기록, 배포 산출물 또는 캐시에 남을 수 있으므로 커밋 전에 아래 기준을 확인한다.

## 공개 가능

- 사이트에 표시할 이름과 개발자 소개
- 공개 연락처로 선택한 이메일
- GitHub 및 Instagram 공개 프로필
- 최적화하고 워터마크를 적용한 공개용 사진
- 투자 종목별 비중, 상대 지수, 공개하기로 한 투자 의견
- 도구 소스와 일반적인 테스트 데이터

## 저장소에 올리지 않음

- API 키, 액세스 토큰, 비밀번호, 쿠키와 세션 값
- 증권사 API 인증서, 앱 키와 시크릿, 계좌번호
- 주민등록번호, 전화번호, 주소, 신분증과 금융 문서
- 투자 총액, 잔액, 수량, 매수금액이 보이는 원본 캡처
- `.env` 파일과 개인용 설정 파일
- 원본 사진, 위치정보가 남은 사진, 비공개 사진
- 데이터베이스, 백업, 로그, 내려받은 금융 CSV·엑셀 파일

민감한 원본은 저장소 밖에 보관한다. 프로젝트 안에서 임시로 사용해야 한다면 `private/`, `captures/`, `screenshots/`, `data/private/` 중 하나에 두며 이 경로는 Git에서 제외된다.

## 커밋 전 점검

1. `pnpm security:check`를 실행한다.
2. `git status`에서 새 파일을 직접 확인한다.
3. 캡처와 사진에 계좌·금액·위치정보가 남아 있지 않은지 확인한다.
4. 공개 연락처와 공개 프로필 외의 개인정보가 본문에 없는지 확인한다.

배포 과정에서도 민감정보 검사가 자동 실행된다. 검사에서 의심 파일이나 키 형식이 발견되면 배포가 중단된다.

## 실수로 커밋했을 때

파일을 지우는 것만으로는 충분하지 않다. 노출된 키와 토큰은 즉시 폐기·재발급하고, 계정 비밀번호를 변경한 뒤 Git 기록 제거 여부를 판단한다. 공개 저장소에 푸시한 정보는 이미 복제되었을 수 있다고 가정한다.
